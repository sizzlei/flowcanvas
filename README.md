# FlowCanvas

**FlowCanvas**는 플로우차트·아키텍처 다이어그램을 시각적으로(드래그 앤 드롭) 만드는 독립형 에디터입니다. 도형과 흐름선을 캔버스에 그리고, 흐름선에 애니메이션·순서·태그를 지정하고, AWS 아이콘 등 이미지를 자유롭게 배치할 수 있습니다.

빌드 과정도, 서버도, 외부 의존성도 없습니다. `index.html`을 최신 브라우저에서 열면 바로 동작합니다.

## ✨ 기능

- **드래그로 그리기** — 상단 팔레트의 도형을 캔버스로 끌어 배치하면 바로 노드가 됩니다.
- **도형 8종** — 둥근 사각형, 사각형, 스타디움, 마름모, 원, 육각형, 원통(DB), 서브루틴.
- **드래그로 연결** — 노드에 마우스를 올리면 나오는 점을 다른 노드로 끌어 흐름선을 만듭니다.
- **흐름선 스타일** — 실선/점선/굵게, 끝모양은 화살표/없음/원/X/양방향.
- **흐름선 라우팅** — 곡선/직선 전환(`Cmd/Ctrl+E`), 흐름선별 **직각(ㄱ자) 연결**, 선 위를 드래그해 만드는 **핀포인트(꺾임점)**. Shift 드래그로 45°/90° 각도 잠금.
- **흐름선 애니메이션** — 흐름선을 따라 빛이 흐르는 효과를 순서대로 재생하고 GIF/WebM으로 내보내기.
- **태그** — 흐름선을 태그로 묶어 원하는 것만 표시/숨김.
- **그룹(서브그래프)** — 노드를 묶어 제목 있는 박스로 만들고, 박스째 드래그로 이동. 더블클릭으로 이름 변경.
- **컨테이너** — VPC·Region·Subnet 같은 크기 조절 박스로 노드를 담기(AWS 프리셋 제공).
- **AWS 아이콘 & 이미지** — 내장 AWS 아키텍처 아이콘 검색·배치, 이미지 붙여넣기/드래그&드롭.
- **영역 선택** — 빈 곳을 드래그해 여러 노드를 한 번에 선택하고 이동·삭제·색상·그룹화.
- **자동 줄바꿈** — 긴 라벨은 여러 줄로 감기고 노드 크기가 자동 조절됩니다.
- **노드 스타일** — 채움 색, 테두리 색·스타일(실선/점선/굵게).
- **격자 스냅** — 드래그·방향키 이동을 20px 격자에 맞춤.
- **캔버스 이동/확대** — `Space + 드래그`로 이동, 휠로 확대/축소, `F`로 화면 맞춤, 상단 배율 컨트롤 제공.
- **실행취소/다시실행** — `Cmd/Ctrl+Z`, `Cmd/Ctrl+Shift+Z`.
- **우클릭 메뉴** — 노드·흐름선·그룹·컨테이너·빈 곳에서 상황에 맞는 작업을 바로 실행.
- **내보내기** — **PNG** 이미지, **애니메이션 GIF/WebM**, **`.yaml`** 프로젝트 파일로 저장(파일명 지정 가능).
- **저장/불러오기** — 작업 내용이 브라우저에 자동 저장되어 다시 열어도 복원되고, `.yaml`로 내보내 다른 기기로 옮길 수 있습니다(레거시 `.json`도 열림).
- **단축키 설정** — 모든 단축키를 설정에서 재지정할 수 있고, 설정은 JSON으로 저장됩니다.

## 📁 프로젝트 구조

```
flowcanvas/
├── index.html      # 마크업 (툴바, 캔버스, 설정 모달)
├── styles.css      # 스타일 (다크 테마, :root의 CSS 변수로 관리)
├── app.js          # 로직 (무의존성) — 상단 주석에 섹션 지도 있음
├── aws-icons.js    # 내장 AWS 아키텍처 아이콘 데이터
├── guide.js        # 인앱 사용자 가이드(GUIDE.md에서 생성)
├── gif.js          # GIF 인코더
├── confluence-forge/ # Confluence Cloud용 Forge 래퍼(선택)
├── README.md
└── LICENSE
```

`app.js` 맨 위에는 뷰/팬·줌, 상태, 렌더링, 선택, 상호작용, CRUD, 저장 트리거, PNG 내보내기, 직렬화, 히스토리, 단축키, 시작 등 섹션을 설명하는 주석이 있습니다.

## 🚀 사용 방법

### 방법 A — 그냥 열기
폴더를 받아 `index.html`을 더블클릭하세요.

### 방법 B — GitHub Pages로 무료 배포
1. GitHub에 새 저장소를 만들고 **모든 파일**(`index.html`, `styles.css`, `app.js`, `aws-icons.js`, `guide.js`, `gif.js` 등)을 올립니다.
2. `main` 브랜치에 커밋/푸시합니다.
3. 저장소 **Settings → Pages**로 이동합니다.
4. **Build and deployment → Source**에서 **Deploy from a branch**를 선택합니다.
5. 브랜치 **`main`**, 폴더 **`/ (root)`**를 지정하고 **Save**를 누릅니다.
6. 1~2분 뒤 `https://<사용자명>.github.io/<저장소명>/`에서 앱이 열립니다.

정적 사이트(HTML + CSS + JS)라 별도 빌드나 설정 없이 그대로 서빙됩니다.

## ⌨️ 기본 단축키

| 동작 | 단축키 |
| --- | --- |
| 선택 삭제 | `Delete` / `Backspace` |
| 실행취소 | `Cmd/Ctrl + Z` |
| 다시실행 | `Cmd/Ctrl + Shift + Z` |
| 곡선/직선 전환 | `Cmd/Ctrl + E` |
| 화면 맞춤 | `F` |
| 파일 저장 | `Cmd/Ctrl + S` |
| 파일 열기 | `Cmd/Ctrl + O` |
| 선택 해제 | `Esc` |
| 화면 이동 | `Space` + 드래그 |
| 영역 선택 | 빈 캔버스에서 드래그 |

이동/영역선택을 제외한 모든 단축키는 ⚙ 설정에서 바꿀 수 있습니다.

## 🗺️ 데이터 저장 방식

- **자동 저장**: 변경할 때마다 현재 다이어그램이 `localStorage`(`flowcanvas.diagram`)에 저장되어 자동 복원됩니다.
- **설정**: 단축키 설정은 `localStorage`(`flowcanvas.settings`)에 JSON으로 저장됩니다.
- **파일**: **💾 저장**으로 받은 `.yaml`은 커밋·공유하거나 **📂 열기**로 다시 불러올 수 있습니다(레거시 `.json`도 열림).

## 🛠️ 기술

SVG 캔버스를 사용하는 순수 HTML/CSS/JavaScript. 프레임워크 없음, 외부 네트워크 호출 없음 — 완전히 오프라인에서 동작합니다.
